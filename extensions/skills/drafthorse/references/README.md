# Authoring Guides

These three files — `collecting-references.md`, `step-splitting.md`, `condition-writing.md` — are the phase-specific judgment the DraftHorse skill loads when a step reaches for it: how to harvest and classify references, where to cut step edges, how to write conditions that carry the routing.

They are deliberately kept as **references**, not reshaped into steps or handover docs, even though they read as guidance rather than inert data. Two reasons:

- **Not all references need converting.** The DraftHorse framework's *references carry data* rule is conceptual, not a hard law — the interpreter is an agent, and a phase guide is data the drafting agent *reads to decide with*, not a procedure it *executes in order*. Each is cited at one moment of use by the step that needs it (`collecting-references.md` from `+Collect References`, and so on), which is exactly the reference contract.
- **They are a trilogy, one per authoring phase.** Keeping them as a matched set of sibling files — same shape, same role — is clearer than folding each into its step or promoting each to a handover doc. The parallel structure is the point.

A spec-checker (drafthorse-saddler) may flag these under the *data, not work* reference check; that is a known, accepted call recorded here, not an unaddressed defect.
