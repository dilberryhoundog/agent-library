# Students

One file per learner lives here — the configuration layer the classroom skill reads to
tailor every build (identity, learning profile, interests, current levels and goals, format
preferences, worldview/content settings, resources on hand, prior-build history).

To add a learner, ask the classroom skill to create one: it reads the field definitions in
the skill's `references/students/_template.md` and writes `students/<name>.md` here. You can
also copy that template yourself and fill it in.
